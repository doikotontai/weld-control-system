WELD CONTROL ONLINE - HƯỚNG DẪN TRIỂN KHAI
=========================================

1. MỤC TIÊU
- Deploy bản preview/prod lên Vercel.
- Kết nối đúng Supabase project.
- Không để thiếu env và không để app build lên nhưng runtime mới vỡ.

2. BIẾN MÔI TRƯỜNG BẮT BUỘC TRÊN VERCEL
- NEXT_PUBLIC_SUPABASE_URL
- NEXT_PUBLIC_SUPABASE_ANON_KEY
- SUPABASE_SERVICE_ROLE_KEY

Nguồn lấy:
- Supabase Dashboard -> Project Settings -> API

Ghi chú:
- NEXT_PUBLIC_* được expose cho browser.
- SERVICE_ROLE_KEY chỉ dùng server-side, phải đặt trên Vercel và không commit vào repo.

3. THỨ TỰ CHẠY SQL TRÊN SUPABASE
Nếu là project mới:
- Chạy `setup_database.sql`

Nếu database đã tồn tại:
- Backup database trước.
- Chạy thẳng `migration_add_weld_columns.sql`.
- Redeploy app ngay sau khi SQL xong.
- Không chạy lại mù quáng `schema.sql` nếu đã có bảng/dữ liệu production.
- Xem thêm `HUONG_DAN_MIGRATION_DB_TRUC_TIEP.md`.

4. KIỂM TRA SAU KHI CHẠY SQL
- Bảng `welds` có đầy đủ các cột app đang dùng.
- Login được.
- Dashboard load được dữ liệu.
- Import Excel không báo lỗi schema.
- Tạo request / sửa weld lưu được.

5. DEPLOY LÊN VERCEL
Preview:
- Import repo vào Vercel hoặc deploy bằng CLI.
- Framework: Next.js
- Root Directory: `huongtt/weld-control` nếu repo root cao hơn.
- Build Command: `npm run build`
- Output: mặc định của Next.js

Production:
- Sau khi preview OK mới promote/production deploy.

6. VERCEL SETTINGS CẦN CHECK
- Environment Variables: đủ 3 key Supabase cho Preview và Production.
- Thực tế hiện tại: Production + Development đã được sync bằng CLI; Preview nên thêm tay trên Vercel Dashboard nếu muốn dùng link preview có login hoàn chỉnh.
- Node version: để Vercel tự nhận theo Next.js 16, nếu cần có thể set Node 20.
- Root Directory đúng với source app.

7. SUPABASE SETTINGS CẦN CHECK
- Auth -> URL Configuration:
  - Site URL = domain Vercel production
  - Additional Redirect URLs = preview URL nếu cần login ở preview
  - Nếu preview đổi domain mới, thêm domain preview mới vào đây
- API keys đúng project đang chạy thật.
- SQL Editor đã apply các migration cần thiết.

8. NẾU LOGIN TRÊN VERCEL BỊ LỖI
Kiểm tra theo thứ tự:
- Sai NEXT_PUBLIC_SUPABASE_URL
- Sai NEXT_PUBLIC_SUPABASE_ANON_KEY
- Sai SUPABASE_SERVICE_ROLE_KEY
- Chưa thêm domain Vercel vào Supabase Auth URL config
- Database chưa có schema/migration mới

9. CHECKLIST CHỐT TRƯỚC KHI BÀN GIAO
- `npm run lint` pass
- `npm run build` pass
- Login được
- Dashboard / Requests / Welds / Import / Admin mở được
- Console browser không có error runtime
- Đã backup hoặc ghi nhận các test data đã tạo trên Supabase
